## 4. 数据流架构

### 4.1 数据读取流程

```
用户请求
    ↓
Next.js Server Component
    ↓
Supabase Client (服务端)
    ↓
PostgreSQL Database
    ↓
RLS 权限检查
    ↓
返回数据
    ↓
Server Component 渲染
    ↓
HTML 返回给用户
```

### 4.2 数据写入流程

```
用户操作 (Client Component)
    ↓
表单验证 (Zod Schema)
    ↓
API Route / Server Action
    ↓
权限验证 (JWT Token)
    ↓
Supabase Client
    ↓
PostgreSQL Database
    ↓
RLS 权限检查
    ↓
数据写入
    ↓
触发器执行 (如有)
    ↓
返回结果
    ↓
UI 更新 (React Query)
```

### 4.3 文件上传流程

```
用户选择文件
    ↓
前端验证 (大小、格式)
    ↓
API Route (/api/upload)
    ↓
Supabase Storage API
    ↓
文件存储到 S3
    ↓
返回文件 URL
    ↓
保存 URL 到 media 表
    ↓
返回给前端
```

## 5. 认证授权架构

### 5.1 认证流程

```
┌─────────────────────────────────────────────────────────────┐
│                      认证流程                                │
├─────────────────────────────────────────────────────────────┤
│  邮箱密码登录:                                               │
│  1. 用户输入邮箱密码                                         │
│  2. Supabase Auth 验证                                      │
│  3. 返回 JWT Token                                          │
│  4. Token 存储到 Cookie                                     │
│  5. 后续请求携带 Token                                      │
├─────────────────────────────────────────────────────────────┤
│  OAuth 登录 (Google/GitHub):                                │
│  1. 用户点击第三方登录                                       │
│  2. 跳转到 OAuth 提供商                                     │
│  3. 用户授权                                                │
│  4. 回调到应用                                              │
│  5. Supabase 创建/更新用户                                  │
│  6. 返回 JWT Token                                          │
│  7. Token 存储到 Cookie                                     │
└─────────────────────────────────────────────────────────────┘
```

## 6. 缓存策略

### 6.1 多层缓存架构

```
┌─────────────────────────────────────────────────────────────┐
│  L1: 浏览器缓存                                              │
│  • 静态资源 (JS, CSS, 图片)                                  │
│  • Cache-Control: public, max-age=31536000                  │
├─────────────────────────────────────────────────────────────┤
│  L2: CDN 缓存 (Vercel Edge)                                 │
│  • 静态页面                                                  │
│  • 图片优化后的资源                                          │
│  • Cache-Control: s-maxage=3600, stale-while-revalidate    │
├─────────────────────────────────────────────────────────────┤
│  L3: Next.js 缓存                                            │
│  • Server Component 缓存                                    │
│  • Data Cache (fetch 缓存)                                  │
│  • revalidate: 3600 (1小时)                                 │
├─────────────────────────────────────────────────────────────┤
│  L4: React Query 缓存                                        │
│  • 客户端数据缓存                                            │
│  • staleTime: 5 * 60 * 1000 (5分钟)                         │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 缓存失效策略

```typescript
// 文章发布/更新时
revalidatePath('/blog')
revalidatePath(`/blog/${slug}`)

// 项目发布/更新时
revalidatePath('/projects')
revalidatePath(`/projects/${slug}`)

// 首页数据更新时
revalidatePath('/')
```

## 11. 扩展性设计

### 11.1 水平扩展

- **前端**: Vercel 自动扩展，无需配置
- **数据库**: Supabase 支持垂直扩展，可升级实例
- **存储**: Supabase Storage 自动扩展

### 11.2 功能扩展

- **评论系统**: 集成 Giscus (GitHub Discussions)
- **搜索优化**: 集成 Algolia 全文搜索
- **邮件通知**: 集成 SendGrid / Resend
- **分析增强**: 集成 Plausible / Umami
